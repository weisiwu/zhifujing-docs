# AI 生成 APP 设计稿最佳实践

## 概述

AI 生成 UI 的核心矛盾与破解方案。

## 核心矛盾

| 问题 | 说明 |
|------|------|
| 文生图不懂层级 | AI 生成的图片缺少组件层级关系，无法直接转为代码 |
| 大模型控不了像素 | 纯文本 Prompt 无法精确控制像素级布局 |

## 破解方案

**组件库 + 大模型组合架构**：
- 让大模型做「组合决策」而非「像素绘制」
- 基于组件库的约束，保证输出的可实现性
- 通过 Schema/JSON 描述布局，而非直接生成图片

## 工具对比

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| Pixso AI | 国内主流，组件库丰富 | 企业级设计 |
| v0 Dev | Vercel 出品，直接生成代码 | 快速原型 |
| Figma AI | 设计工具原生集成 | 设计师工作流 |
| Galileo AI | 文本到 UI | 早期探索 |
| Uizard | 线框图到高保真 | 产品经理 |
| Locofy | 设计稿转代码 | 前端开发 |
| Magician (Figma 插件) | 辅助设计 | 日常设计 |

## 最佳实践

1. 先定义组件库约束，再让 AI 做组合
2. 用 JSON/Schema 描述布局而非自然语言描述像素
3. AI 输出 → 人工审查 → 组件库映射 → 代码生成
