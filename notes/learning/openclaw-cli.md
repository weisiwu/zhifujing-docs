# 如何用 OpenClaw CLI 强化 Agent

## 概述

把 OpenClaw CLI 理解为 Agent 的「控制平面」，通过六条路径强化 Agent 能力。

## 六条强化路径

| # | 路径 | 说明 |
|---|------|------|
| 1 | 重做拓扑 | 调整 Agent 间的关系和数据流 |
| 2 | 补知识/流程 | 通过 Skills 注入领域知识和 SOP |
| 3 | 接入工具 | 通过 Built-in/Plugins 扩展工具集 |
| 4 | 配置模型/权限 | 调整模型选择和能力边界 |
| 5 | 启用 Hooks | 事件驱动的自动化响应 |
| 6 | MCP 桥接 | 通过 MCP 连接更大的 Agent 网络 |

## 核心理念

> CLI 不是「命令行工具」，而是 Agent 的「控制平面」。通过 CLI 可以重新编排 Agent 的能力拓扑，实现从单 Agent 到多 Agent 网络的升级。

## 实践要点

1. 先理清拓扑，再选择强化路径
2. Skills 是最高性价比的强化方式
3. MCP 适合需要外部系统能力的场景
4. Hooks 适合做自动化响应和审计
